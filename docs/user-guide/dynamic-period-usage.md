# 动态评分周期 - 用户使用手册

本手册面向系统管理员，介绍如何使用动态评分周期功能。

## 📖 功能概述

动态评分周期功能允许管理员根据实际需要，灵活地在**单周（7天）**和**双周（14天）**两种评分周期之间切换，无需重启系统或修改数据库。所有历史评分数据都会被完整保留。

## 👥 用户角色

| 角色 | 权限 |
|------|------|
| 管理员 | 可以查看和修改周期配置 |
| 教师/学生 | 只能查看当前周期信息 |

## 🎯 核心功能

### 1. 查看当前周期配置

**访问路径：** 管理员面板 → 学期设置

在学期设置页面，您可以看到：
- **当前周期类型**：显示是单周还是双周
- **评分周期配置卡片**：展示详细的周期信息

### 2. 变更周期类型

#### 操作步骤：

1. 登录管理员账户
2. 进入 **学期设置** 页面
3. 找到 **评分周期配置** 区域
4. 选择新的周期类型：
   - **双周 (14天)** - 适合常规学期
   - **单周 (7天)** - 适合密集评分期或特殊安排
5. 选择生效日期（必须是未来日期）
6. 填写变更原因（可选，建议填写以便追溯）
7. 点击 **变更周期类型** 按钮
8. 在确认对话框中再次确认操作

#### 示例场景：

**场景1：学期开始阶段**
- 问题：新学期开始，希望每周都能及时发现问题
- 解决方案：将周期类型从双周改为单周
- 操作：选择"单周 (7天)"，生效日期选择下周一

**场景2：学期中期调整**
- 问题：评分频率太高，师生负担重
- 解决方案：从单周改回双周
- 操作：选择"双周 (14天)"，生效日期选择两周后

### 3. 查看配置历史

点击 **查看配置历史** 按钮，可以查看所有的周期类型变更记录：
- 变更时间
- 变更内容（从什么类型改为什么类型）
- 生效日期和周期号
- 操作人
- 变更原因

## ⚠️ 重要注意事项

### 数据完整性保证

✅ **历史数据不会被修改**
- 变更周期类型不会影响任何已提交的评分记录
- 所有历史周期的信息都会被永久保存
- 统计报表会正确反映每个周期的实际长度

✅ **无缝过渡**
- 系统会自动处理周期切换
- 新旧周期之间不会有数据丢失
- 学生和教师无需任何额外操作

### 变更限制

❌ **不能回溯**
- 生效日期必须是未来日期
- 不能修改已经开始的周期

❌ **频繁变更**
- 建议不要频繁变更周期类型
- 每次变更前请充分考虑影响

## 📊 常见问题 (FAQ)

### Q1: 变更周期类型后，当前周期会立即改变吗？

**A:** 不会。系统会在您指定的生效日期之后的下一个周期开始应用新的周期类型。当前正在进行的周期不会受到影响。

### Q2: 如果我设置了单周，能不能中途改回双周？

**A:** 可以。您随时可以变更周期类型，只要指定未来的生效日期即可。

### Q3: 历史评分数据会受影响吗？

**A:** 完全不会。所有历史评分数据都保留其原始的周期信息，不会被修改或重新计算。

### Q4: Excel导出报表会正确显示不同长度的周期吗？

**A:** 会的。导出的Excel报表会准确反映每个周期的实际长度（7天或14天），统计数据也会相应调整。

### Q5: 学生和教师能看到周期配置变更吗？

**A:** 他们可以看到当前是单周还是双周，但不能修改配置。配置变更只有管理员可以执行。

### Q6: 如果忘记了某次变更的原因，能查到吗？

**A:** 可以。在"配置变更历史"中可以查看所有变更记录，包括变更原因（如果当时填写了的话）。

## 💡 最佳实践

### 1. 规划周期策略

建议在学期开始前规划好周期策略：
- 开学第1-4周：单周（7天）- 快速发现问题
- 第5-14周：双周（14天）- 常规节奏
- 期末前2周：单周（7天）- 加强管理

### 2. 及时沟通

变更周期类型前，建议：
- 提前通知教师和学生
- 说明变更原因和预期效果
- 在系统中记录详细的变更原因

### 3. 定期回顾

每学期结束后：
- 回顾周期配置历史
- 评估不同周期长度的效果
- 为下学期制定更合理的策略
